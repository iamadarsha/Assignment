## рЯОѓ The Problem

Every business day, five Indian and global regulators publish circulars:

```
рЯП¶ RBI        вЖТ Reserve Bank of India  
рЯПЫпЄП IFSCA      вЖТ International Financial Services Centres Authority  
рЯПҐ MCA        вЖТ Ministry of Corporate Affairs  
рЯУИ SEBI       вЖТ Securities and Exchange Board of India  
рЯМР FATF       вЖТ Financial Action Task Force  
```

Most circulars are noise. A handful change how a remittance company has to operate вАФ **tomorrow morning**. Without a system like this, your compliance team spends hours manually sifting PDFs to find the two that actually matter.

**Regulatory Intel fixes that.**

### рЯФЦ Compliance Workflow
- Review queue filtered by `HIGH` / `MEDIUM` / `LOW`
- Persisted reviewed state across sessions (SQLite-backed)
- Source health badges вАФ `ok` / `empty` / `blocked` / `error` per regulator
- Deduplication via `INSERT OR IGNORE` on URL вАФ zero duplicate circulars, ever

## рЯІ† How It Works

```mermaid
flowchart LR
    A["5 Regulator Sources\nRBI ¬Ј IFSCA ¬Ј MCA ¬Ј SEBI ¬Ј FATF"] --> B["Parallel Fetcher\nfail-isolated"]
    B --> C[("SQLite\ncirculars.db")]
    C --> D{PDF?}
    D -->|yes| E["pdf-parse вЖТ OCR fallback"]
    D -->|no| F["HTML extraction"]
    E --> G["Structured Chunks\nsection + page metadata"]
    F --> G
    G --> H["Gemini 2.5 Flash\nrelevance + actions"]
    H -->|fails| I["Groq LLaMA 3.3\nfallback"]
    H --> J["UI: scored cards + Q&A"]
    I --> J

    style A fill:#3b82f6,stroke:#fff,color:#fff
    style H fill:#f59e0b,stroke:#fff,color:#fff
    style I fill:#f59e0b,stroke:#fff,color:#fff
    style J fill:#22c55e,stroke:#fff,color:#fff
```

**Three steps to compliance clarity:**
1. **Fetch Updates** вЖТ pulls latest circulars from all 5 sources in parallel
2. **Process AI** вЖТ runs Gemini analysis (auto-loops until every circular is scored)
3. **Filter вЖТ Review вЖТ Ask** вЖТ work only the `HIGH` signals, interrogate the source document with citations

## вЪ° Quick Start

### Prerequisites

- Node.js 18+
- `GEMINI_API_KEY` вАФ [get one free вЖТ](https://aistudio.google.com/app/apikey)
- `GROQ_API_KEY` вАФ [get one free вЖТ](https://console.groq.com)

### Install & Run

```bash
# Clone the repo
git clone https://github.com/iamadarsha/Finance-News-Analyser.git
cd Finance-News-Analyser

# Install dependencies
npm install

# Add your API keys
cat <<EOF >> .env.local
GEMINI_API_KEY=your_gemini_key_here
GROQ_API_KEY=your_groq_key_here
EOF

# Start the dev server
npm run dev
# вЖТ http://localhost:3000
```

### Your First 3 Minutes

```
 Step 1  вЖТ  Click "Fetch Updates"     Pulls all 5 sources in parallel
 Step 2  вЖТ  Click "Process AI"        Gemini scores every circular
 Step 3  вЖТ  Filter to HIGH            Only the ones that matter
 Step 4  вЖТ  Expand any card           Read action items + evidence
 Step 5  вЖТ  "Ask This Document"       Interrogate the PDF with citations
 Step 6  вЖТ  Mark as Reviewed         Clear the queue
```

## рЯЪҐ Deployment

### Vercel (One Command)
```bash
npx vercel
# Add GEMINI_API_KEY + GROQ_API_KEY in the Vercel dashboard
```
> SQLite data is ephemeral on Vercel's filesystem. See [ARCHITECTURE.md](./ARCHITECTURE.md) for the Postgres migration path.

### Self-Hosted Node.js
```bash
npm run build && npm start
# Data persists in ./data/circulars.db вАФ back this up or mount as a volume
```

### Docker
```bash
docker build -t regulatory-intel .
docker run -p 3000:3000 \
  -e GEMINI_API_KEY=your_key \
  -e GROQ_API_KEY=your_key \
  -v $(pwd)/data:/app/data \
  regulatory-intel
```

### Fly.io
A `fly.toml` is included for persistent-volume deploys вАФ ideal for keeping circular history across restarts.
